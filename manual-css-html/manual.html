Application manual

Texinfo is a typesetting syntax used for producing documentation for software manuals. It’s the official documentation format for GNU projects.

This template contains sections to test CSS style as well as example of code with syntax highlighting applied in some examples.

It is a variant from the classical GNU manual CSS style available at:

  • https://www.gnu.org/software/gnulib/manual.css
  • https://www.gnu.org/style.css

1 Usage

To apply the style sheet to the HTML version of your documentation, run the makeinfo command specifying the appropriate url to the style sheet using the --css-ref option.

--css-ref=url adds to each output HTML file a <link> tag referencing a CSS at the given url.

For example: 

$ makeinfo --html --css-ref=URL -o OUTPUT_DEST  FILE.texi

Some example values would be:

  1. A URL to the style sheet in an external domain: 
    https://mydomain/css/manual.css
  2. An absolute or relative URL path to the style sheet in the same domain where the documentation is published. 
    manual.css
/manual.css
/css/manual.css
static/css/manual.css

For OUTPUT_DEST, it depends if the produced HTML will be split or not:

With split output:

creates DEST as a directory and put the output files there.

With non-split output:

if DEST is already a directory or ends with a /, put the output file there. Otherwise, DEST names the output file.

For example, to build the a manual run the following command: 

$ makeinfo --html --css-ref=manual.css --no-split -o manual.html  manual.texi

2 Elements

  • Inline Elements

2.1 Inline Elements

Inline elements look like this:

  • @code: A text that is a piece of a program: printf, while, if, return.
  • @kbd: Command for characters of input to be typed by users, eg: Ctrl-x + s, Ctrl-x Ctrl-f
  • @key: eg: F1, F2, DEL, PGUP
  • @samp: Sample text ‘yyyy-dd-mm’
  • @verb: Inline verbatim: 2 + 3
  • @var: variables: my_struct, record.
  • @env: PATH.
  • @file: ~/.emacs/, /home/.
  • @command: ls -l
  • @option: makeinfo --css-ref
  • @dfn: to introduce a term: main, blacklist, primary, whitelist
  • @abbr: Mr. (Mister), Mrs. (Missus) or Ms. (Missus)
  • @acronym: GNU (GNU’s Not Unix), ASAP (as soon as possible) project.
  • @indicateurl: great for url which doesn’t exist ‘https://doesnot-exist.com/’.
  • @email: no-reply@mailbox.org.
  • @emph: emphasized.
  • @strong: strong
  • @sc: SMALL CAPS FONT

2.2.1 @quotation

In Texinfo, the @quotation command is used for quotations, and for information messages—why is that? I don’t know. The former looks like this:

Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat.

— Someone famous in Source Title

Information messages such as notes, tips, warnings, etc., look like this:

Note: Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat.

Extending the texinfo with a Macro, 

@macro admonition {keyword}
@ifhtml
@html
<div class=\keyword\>
@end html
@end ifhtml
@end macro

@macro endadmonition {}
@ifhtml
@html
</div>
@end html
@end ifhtml
@end macro

it is possible to have standard Notes, Warning, Important admonition, eg:

Note: Class aptent taciti sociosqu ad litora torquent per conubia nostra, per inceptos himenaeos. Quisque dapibus ipsum ac semper ultricies. Nunc eget est convallis, rhoncus ex id, sagittis dolor. Aliquam ac dignissim arcu. Nunc elit est, sollicitudin id enim nec, efficitur aliquam nibh. Maecenas aliquam tempor libero quis commodo. Morbi nisl lectus, fermentum in diam ut, fringilla sollicitudin ligula.

Warning: Vestibulum mattis, nunc at venenatis pharetra


Important: Nulla semper lobortis elit, nec gravida velit lacinia nec. Nulla fringilla ex ut lorem hendrerit cursus. Praesent quis imperdiet velit. Donec semper tortor purus, vehicula pharetra augue blandit a. Mauris vitae aliquam magna.

3.1 Introduction

Source code is highlighted with an external program: GNU Source-highlight link.

GNU Source-highlight reads the source file and convert it to a file ready to be included within the documentation.

  • Pros / Cons

3.1.1 Pros / Cons

Pros:

  • Better code clarity
  • Fully automatic (with the proper Makefile.
  • Covers a large range of language and output formats which suits Texinfo

Cons:

  • Complexifies the Makefile
  • Makes the documentation dependent from another external program.
  • For a nicer representation different output formats need to be generated. Although there is the possibility to only produce the texinfo version.

3.2 standard code - reference

This is how the source code would look like by default without the syntax being highlighted: 

#include <stdio.h>
#include <math.h>               /* NAN, isnan, nan */

typedef struct record_s {
    char *name;
    double data[20];
} record_s;

static void
print_record (record_s *r)
{
  printf ("The name is: %s\n  ", r->name);
  for (size_t i=0; !isnan (r->data[i]); i++) printf ("%g  ", r->data[i]);
  printf ("\n");
}

int
main (int argc, char *argv[])
{

  record_s r1 = (record_s){.name="Paul", .data={1, 2, 3, NAN}};
  record_s r2 = (record_s){.name="John", .data={4.1, 5, 6, 7.2, NAN}};

  print_record (&r1);    /* ❶ comment 1 */
  print_record (&r2);

  return 0;
}
